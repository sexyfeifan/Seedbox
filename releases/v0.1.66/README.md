# Seedbox

一个偏向“小红书内容私藏”的自部署收藏箱：抓链接、抓正文、抓图片、抓视频，并尽量把内容以更稳定、更适合长期保存的形式留在你自己的服务里。

## 1. 软件介绍

- Seedbox 面向“收藏、回看、整理、私藏”场景，尤其适合收集小红书内容。
- 它不是单纯的书签工具，而是把正文、图片、视频、作者信息、平台信息一起结构化保存下来。
- 当前版本重点放在：`小红书内容抓取能力`、`多平台兼容抓取`、`网页端浏览与编辑`、`服务端自部署`。

## 2. 核心能力

- 更好地私藏小红书内容：支持正文、图文混排、视频、Live 图混合内容的抓取与展示。
- 稳定保存媒体内容：针对图片与视频做素材回传、缓存、下载与展示兜底。
- 支持常见中文内容平台：当前已覆盖小红书、微博、抖音、豆瓣、知乎、Bilibili 等站点的抓取链路。
- 网页端直接使用：支持桌面浏览、详情查看、原位编辑、重新解析、批量归档。
- 服务端自托管：支持 Docker / fnOS 场景，适合长期持久化运行。

## 3. 适合什么场景

- 收藏小红书旅行攻略、穿搭、摄影、探店、灵感内容，避免后续失效或被删。
- 把微博、抖音、豆瓣等内容统一收进自己的资料库。
- 做个人灵感库、选题库、素材库、长期稍后阅读清单。
- 自己掌控数据，不依赖第三方收藏平台的在线可用性。

## 5. 当前技术结构

- 客户端：`Flutter`（iOS / Android / Web / Desktop，当前正式发布以服务端为主）
- 服务端：`Node.js + Fastify`
- 解析引擎：`Readability + Playwright + 站点规则系统`
- 存储模式：开发环境支持内存持久化，生产可切换数据库模式
- 部署方式：`Docker Compose / fnOS`

## 6. 现在已经能做什么

- 抓取分享链接并自动提取真实 URL
- 解析正文、标题、作者、发布时间、地点标签
- 识别并保存图片、视频、封面、头像、站点图标
- Web 端查看收藏列表、详情、媒体画廊
- 原位编辑收藏标题、标签、状态并支持重新解析
- 支持服务端统一发版、DockerHub 推送与 compose 产物生成

## 7. 部署与测试

- 参考 `docs/local-testing.md`
- 自建落地参考：`docs/self-hosting.md`
- Docker 部署与发布：`docs/docker-deploy.md`
- 一键烟雾测试：`./scripts/smoke-test.sh`
- 摘要专项烟雾：`./scripts/summary-smoke-test.sh`
- 计费专项烟雾（默认关闭）：`ENABLE_BILLING_SMOKE=1 ./scripts/billing-smoke-test.sh`
- 多站点解析回归：`./scripts/parser-regression.sh`
- 全量回归（烟雾 + 解析 + 重复收藏）：`./scripts/full-regression.sh`
- 历史噪音资源清理（可选）：`./scripts/cleanup-noise-assets.sh`

最短路径（无 Docker）：

1. `cd apps/backend && npm install && npm run start:memory`
2. 新终端：`cd apps/parser_worker && npm install && npm run start:api`
3. 新终端：`./scripts/smoke-test.sh`
4. 浏览器打开 `http://localhost:3000/app` 使用网页端

可选安全加固（公网建议）：

1. 后端加 `CLIENT_ACCESS_TOKEN=<your-client-token>`
2. 脚本加 `CLIENT_TOKEN=<your-client-token>`
3. Web 端在浏览器 Console 执行 `localStorage.setItem("seedbox_client_token","<your-client-token>")` 后刷新

自动化脚本：

1. 一键起停开发栈：`./scripts/dev-stack.sh up|down|status`
2. 一次性自动回归：`./scripts/autopilot.sh once`
3. 持续循环回归：`./scripts/autopilot.sh loop`
4. Flutter 平台工程初始化：`./scripts/bootstrap-mobile-platforms.sh`
5. fnOS 版本留存打包：`./scripts/release-fnos.sh v0.1.33`（会新增 `releases/v0.1.33` 与对应 tar 包，不覆盖历史版本）
6. 历史噪音资源排查：`DRY_RUN=1 ./scripts/cleanup-noise-assets.sh`
7. 服务端版本校验（backend + parser + compose）：`./scripts/check-unified-version.sh`
8. 服务端统一发布（DockerHub + compose）：`./scripts/release-unified.sh v0.1.58`

统一发布规则（当前）：

1. 暂停移动端版本发布，正式版本仅发布服务端（backend/parser）。
2. 每次正式发布必须走统一脚本：推送 DockerHub、生成 `releases/vX.Y.Z/docker-compose.yml`。
3. `compose` 仅保留服务端相关版本变量，不再包含移动端版本字段。

移动端 Share 通道接入说明：

1. 见 `docs/mobile-share-bridge.md`

网页端已支持：

1. 本地免登录直连（`x-user-id`，默认演示用户）
2. 粘贴整段分享文案自动提取链接（支持小红书文案）
3. 新增收藏支持“识别剪贴板”自动填充链接
4. 列表筛选（未归档/已归档/全部）
5. 详情高亮关键词
6. 归档与恢复
7. 列表多选后的批量归档/批量恢复
8. 详情高亮词按条目本地记忆
9. 清空已归档（永久删除）
10. 详情页异步摘要（生成/轮询状态/结果展示）
11. 解析结果图片素材回传（详情画廊预览）
12. 图片素材下载（后端按需本地缓存）
13. 商业模式开关：服务端启用 `COMMERCIAL_MODE_ENABLED=true` 后显示 Web 登录/订阅入口
14. 账号与订阅弹层：邮箱验证码登录、订阅刷新、开通/取消 Pro
15. 诊断面板展示媒体过滤摘要（噪音图/风控图过滤统计）

## 8. 关于小红书抓取能力

- 支持小红书分享文案自动抽链，不要求只粘标准 URL。
- 支持图文、视频、图文混合、部分 Live 图混合场景。
- 尽量保留正文原有段落结构，减少抓取后内容“压成一坨”的问题。
- 针对小红书素材链路做了图片候选重建、兼容链接回退与缓存兜底。
- 当前版本的软件定位可以概括为：**更适合私藏小红书内容的自部署收藏工具**。

## 9. 当前发布策略

- 当前正式产物以服务端为主，不再默认生成移动端 IPA。
- 每次正式发布统一同步 backend / parser 版本号。
- 发布产物包含 DockerHub 镜像与 `releases/vX.Y.Z/docker-compose.yml`。
